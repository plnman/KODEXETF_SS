# 🎯 V3.1 핵심 가치(시그널 & 무결성) 복구 계획

사용자가 AI의 시그널을 명확히 인지하고, 그 근거를 의심 없이 신뢰할 수 있도록 대시보드 구조를 전면 재조정합니다.

## Proposed Changes

### [Tab 1] 실전 타점 브리핑 (UI/UX 개선 및 핵심 기능 복구)
- **[Layout]** `st.columns(max_tickers)` 폐지 -> 한 줄에 최대 3~4개씩만 배치하여 **종목명 완벽 노출**. (10종목일 경우 4-4-2 배치 등)
- **[Signal Board]** **'AI 매매 시그널 보드'** 신설.
    - 각 종목별 현재 상태: `🟢 매수 가능`, `🟡 보유(Hold)`, `🔴 매도/관망` 시각화.
    - 핵심 정보 노출: **돌파 목표가(Target)**, **변동성 손절가(Hard Stop)** 추가.

### [Tab 3] 알고리즘 무결성 진단 (투명성 강화)
- **[Parameter Audit]** 현재 엔진에서 사용 중인 실시간 파라미터 노출.
    - 불장 K(0.4) / 안정장 K(0.6) 
    - Z-Score 임계값 (상단 1.5, 하단 -1.5)
- **[Consistency Check]** 백테스팅 엔진과 실전 엔진 간의 로직/파라미터 일치 여부를 상시 체크하여 초록색 체크마크 표시.
- **[Sampling Guide]** 특정 주가와 변동성 데이터를 넣었을 때 시그널이 도출되는 수식 과정을 텍스트/공식으로 설명 추가.

## Verification Plan

### Manual Verification
- 10종목 선택 시 종목명이 'KO...'가 아닌 'KODEX 200' 등으로 명확히 보이는지 확인.
- 오늘 날짜 기준으로 어떤 종목이 '매수 조건'에 근접했는지 시그널 보드에서 확인 가능 여부 점검.
- 무결성 탭에서 현재 엔진의 상세 설정값을 육안으로 확인 가능한지 체크.
